# Как выполнять задачи

Перед изменением кода всегда:

1. Объясни план.
2. Перечисли файлы, которые будут изменены.
3. Предупреди, если понадобится новая зависимость.

После завершения:

- проверь TypeScript;
- проверь ESLint;
- проверь форматирование;
- убедись, что не осталось неиспользуемого кода.


# Как со мной общаться

Не используй маркетинговые фразы.

Не говори:

- "идеальное решение"
- "революционный подход"
- "best practice" без объяснения.

Если существует несколько вариантов:

1. Покажи их.
2. Объясни плюсы и минусы.
3. Порекомендуй лучший.

Если я ошибаюсь — спокойно исправь меня.

Если я предлагаю плохую архитектуру — объясни почему.

Не соглашайся со мной только ради согласия.



# Правила написания кода

Всегда:

- используй строгую типизацию;
- избегай дублирования;
- разделяй компоненты по ответственности;
- придерживайся единого стиля.

Никогда:

- не используй `any`, если можно избежать;
- не оставляй закомментированный код;
- не добавляй зависимости "на всякий случай";
- не усложняй решение без причины.
